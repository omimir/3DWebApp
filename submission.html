Submission

The five things the coursework submission needs. The three URLs go in once the project is published.

  1. Packaged project, launched from VS Code

    Vitrine is a static site with no build step. The repository includes .vscode/extensions.json, which recommends the Live Preview extension. Open index.html in VS Code, choose "Show Preview", and the app runs. It can also be served with npx serve . from the project root.

  2. ITS Web Server URL

    http://users.sussex.ac.uk/~or85@sussex.ac.uk The site is static, uses only relative paths and keeps its own copies of the libraries, so the project folder uploads to the ITS web space without any changes.

  3. GitHub codebase URL

    https://github.com/omimir/3DWebApp This repository holds the application code, the styles, the copies of Three.js and Bootstrap, the glTF models and the Blender source files.

  4. Custom GLSL shaders

    The globe has a thin shell mesh around it running a custom ShaderMaterial. Its fragment shader works out a fresnel term from the view direction and the surface normal: small where the surface faces the camera, large where it turns away. That value is used as the alpha of an additively blended colour, so it reads as a faint atmosphere at the globe's edge without touching the globe's own surface texture.

    There is a second piece of custom GLSL in the post-processing: a vignette pass that darkens the frame edges after tone mapping, the way a studio photo falls off towards the corners.

    Post-processing

    The viewer has an optional pipeline built with the Three.js EffectComposer: a render pass, a bloom pass on the bright highlights, tone mapping, the custom vignette, and an FXAA anti-aliasing pass. It is switched on and off from the controls, and the bloom is eased back when the visitor asks for reduced motion.
